Monday: Introduction to Functions

What is a Function?

In C programming, a **function** is a block of code that performs a specific task. It helps in breaking down a large program into smaller, manageable, and reusable modules. Functions promote code reusability and make the program more organized and easier to debug.

  • Modularity: Breaks down complex problems into simpler parts.
  • Reusability: A function can be called multiple times from different parts of the program.
  • Readability: Makes the code easier to understand and maintain.
  • Abstraction: Hides the implementation details from the user.

Function Syntax

The general syntax for defining a function in C is as follows: 


return_type function_name(parameter_list) {
    // body of the function
    // statements;
    // return value; (if return_type is not void)
}
  • return_type: The data type of the value the function returns. Use void if the function does not return any value.
  • function_name: A unique name for the function.
  • parameter_list: A comma-separated list of input parameters (arguments) that the function accepts. Can be empty.
  • function body: Contains the C statements that perform the function's task.

Example: Simple Function

C Code: 

#include <stdio.h>

// Function definition
void greet() {
    // printf() displays the string inside quotation
    printf("Hello from a function!\n");
}

int main() {
    greet(); // Function call
    return 0;
}
Output: 

Hello from a function!

Tuesday: Function Parameters and Return Values

Passing Arguments to Functions

Functions can accept arguments (also known as parameters) that provide data for the function to operate on. These arguments are specified in the function's parameter list during definition.

  • Formal Parameters: Variables declared in the function definition's parameter list.
  • Actual Arguments: Values passed to the function when it is called.

Return Values

A function can return a single value back to the calling code using the return statement. The data type of this returned value must match the return_type specified in the function definition. 


int add(int a, int b) {
    int sum = a + b;
    return sum; // Returns the sum of a and b
}

Example: Function with Parameters and Return Value

C Code: 

#include <stdio.h>

// Function to add two numbers
int addNumbers(int num1, int num2) {
    int sum = num1 + num2;
    return sum;
}

int main() {
    int result = addNumbers(5, 7); // Call function with arguments
    printf("The sum is: %d\n", result);
    return 0;
}
Output: 

The sum is: 12

Wednesday: Function Prototypes and Scope

Function Prototypes (Declarations)

A **function prototype** (or function declaration) tells the compiler about the function's return type, name, and parameters. It's essential to declare a function before it's used if its definition appears after its call in the code.

  • Syntax: return_type function_name(parameter_type_list);
  • Helps the compiler check for correct function calls.
  • Allows functions to be defined in any order in the source file.

// Function prototype
int multiply(int x, int y);

int main() {
    int prod = multiply(10, 20); // Call function
    return 0;
}

// Function definition (can be after main due to prototype)
int multiply(int x, int y) {
    return x * y;
}

Scope of Variables in Functions

Variables declared inside a function have **local scope**, meaning they are only accessible within that function. Variables declared outside any function have **global scope** and can be accessed from anywhere in the program.

  • Local Variables: Created when the function is called and destroyed when the function exits.
  • Global Variables: Persist throughout the program's execution.

Example: Function Prototype and Scope

C Code: 

#include <stdio.h>

// Global variable
int globalVar = 100;

// Function prototype
void demonstrateScope();

int main() {
    int mainLocal = 10; // Local to main
    printf("In main: mainLocal = %d, globalVar = %d\n", mainLocal, globalVar);
    demonstrateScope();
    return 0;
}

void demonstrateScope() {
    int funcLocal = 50; // Local to demonstrateScope
    printf("In demonstrateScope: funcLocal = %d, globalVar = %d\n", funcLocal, globalVar);
    // printf("In demonstrateScope: mainLocal = %d\n", mainLocal); // ERROR: mainLocal is not accessible here
}
Output: 

In main: mainLocal = 10, globalVar = 100
In demonstrateScope: funcLocal = 50, globalVar = 100

Thursday: Recursion and Storage Classes

Recursion

**Recursion** is a programming technique where a function calls itself to solve a problem. It's often used for problems that can be broken down into smaller, self-similar sub-problems.

  • A **base case** is essential to stop the recursion and prevent infinite loops.
  • Each recursive call solves a smaller part of the problem.

// Recursive function to calculate factorial
int factorial(int n) {
    if (n == 0) { // Base case
        return 1;
    } else {
        return n * factorial(n - 1); // Recursive call
    }
}

Storage Classes in C

Storage classes define the scope, lifetime, initial value, and storage location of a variable. The main storage classes in C are auto, register, static, and extern.

  • auto: Default for local variables. Stored in memory, lifetime limited to function execution.
  • register: Suggests storing the variable in a CPU register for faster access (compiler might ignore).
  • static: Preserves the variable's value between function calls. Local static variables retain their value. Global static variables have file scope.
  • extern: Declares a variable that is defined in another file, making it accessible from the current file.

Example: Recursion and Static Variable

C Code: 

#include <stdio.h>

// Recursive factorial function
int factorial(int n) {
    if (n == 0 || n == 1) {
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

// Function with a static variable
void counter() {
    static int count = 0; // Initialized once
    count++;
    printf("Counter: %d\n", count);
}

int main() {
    printf("Factorial of 5: %d\n", factorial(5));

    counter(); // Output: Counter: 1
    counter(); // Output: Counter: 2
    counter(); // Output: Counter: 3
    return 0;
}
Output: 

Factorial of 5: 120
Counter: 1
Counter: 2
Counter: 3

Friday: Function Pointers and Callback Functions

Function Pointers

A **function pointer** is a variable that stores the memory address of a function. This allows you to call functions indirectly, pass functions as arguments to other functions, and store functions in data structures. 


// Declare a function pointer that points to a function
// taking two ints and returning an int
int (*ptrToFunc)(int, int);
  • Useful for implementing callback mechanisms.
  • Enables dynamic selection of functions at runtime.

Callback Functions

A **callback function** is a function that is passed as an argument to another function and is intended to be called by that other function at a later time. They are common in event handling, sorting algorithms, and asynchronous operations.

Example: Function Pointers and Callback

C Code: 

#include <stdio.h>

// Function to be pointed to
int add(int a, int b) {
    return a + b;
}

// Another function to be pointed to
int subtract(int a, int b) {
    return a - b;
}

// Function that takes a function pointer as an argument (callback)
void performOperation(int num1, int num2, int (*operation)(int, int)) {
    int result = operation(num1, num2);
    printf("Result of operation: %d\n", result);
}

int main() {
    // Declare and initialize a function pointer
    int (*opPtr)(int, int);

    opPtr = add; // Point to the add function
    printf("Using function pointer to add: %d\n", opPtr(10, 5));

    opPtr = subtract; // Point to the subtract function
    printf("Using function pointer to subtract: %d\n", opPtr(10, 5));

    printf("\nDemonstrating callback function:\n");
    performOperation(20, 10, add); // Pass add as a callback
    performOperation(20, 10, subtract); // Pass subtract as a callback

    return 0;
}
Output: 

Using function pointer to add: 15
Using function pointer to subtract: 5

Demonstrating callback function:
Result of operation: 30
Result of operation: 10
